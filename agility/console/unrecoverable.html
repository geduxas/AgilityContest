AgilityContest no puede ejecutarse

Posibles Motivos:

  1. El servidor de base de datos no está activo
    • En equipos lentos, o con poca memoria, el servidor de bases de datos
      puede tardar más tiempo de lo normal en arrancar.
  2. El firewall está bloqueando acceso al puerto local 3306.
    • Compruebe que en la barra de tareas no tiene ningún aviso de seguridad.
    • Compruebe asímismo que en el escritorio no hay ninguna ventana pidiendo
      permiso para ejecutar algún programa.
  3. La base de datos está corrompida. Posibles causas:
    • Errores de tiempos de conexión ( equipo demasiado lento o firewall inspeccionando tráfico de red ).
    • Apagado accidental del equipo con AgilityContest en funcionamiento.
    • Carpetas del servidor de bases de datos borradas o modificadas accidentalmente.
    • Version demasiado antigua de la base de datos que el programa no puede actualizar.

La aplicación no puede continuar mientras el servidor de bases de datos no esté activo
y la base de datos de AgilityContesta consistente y accesible.

Procedimiento a seguir:

  • Por si se tratara de un problema de tiempo de arranque pruebe a esperar unos segundos y
    recargar nuevamente la página web pulsando F5.
  • Elimine bloqueos del antivirus: En el caso de Avast o Esset, es incluso preferible
    desinstalar totalmente dicho programa, dejando como único antivirus "Windows Defender".
  • Asegurese de que el antivirus no está analizando el tráfico de red. Esta operacion hace el equipo muy lento,
    y favorece la aparición de errores del tipo "Se ha excedido el tiempo de conexión".
  • Recupere la última copia de seguridad de la capeta c:\AgilityContest\logs.
    Las copias de seguridad son ficheros con el nombre "agility-YYYYMMDD_hhmm.sql"
    siendo YYYYMMDD_hhmm la fecha en que se realizó dicha copia de seguridad.
  • Reinstale desde cero la aplicación. La última versión está siempre disponible en https://wwww.agilitycontest.es/downloads.
  • Reinice la aplicación una vez reinstalada y recuperar el contenido de la base de datos.
  • En el caso de que el operador tenga conocimientos de mysql puede intentar utilizar
    las herramientas de recuperación del servidor de bases de datos.
    Esto puede ser necesario en el caso de que la versión anteriormente instalada de AgilityContest,
    o la copia de la base de datos de AgilityContest fueran demasiado antiguas.
    En caso de no tener éxito tendrá que proceder a la reinstalación.
